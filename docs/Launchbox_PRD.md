## 💬 Chat Page - AI Agent聊天页面 优先级：🌟🌟🌟🌟

1. ** 对话式交互**
2. ** 快捷调用并完成预设“指令”**
    1. ”指令“ - 也就是工作流，比如 一个生成活动策划案 就是一个工作流 - 需要先生成策划案，然后生成mockup/素材图。 可以直接通过说 指令名称调用这些指令
    2. “动作” - 也就是”技能“，比如 生图 就是一个技能；这些技能可以组合成无限种工作流，可以由人工通过规定“动作” - 来制定工作流，或者AI来理解人的意思，再自己设计工作流（类似manus 一种会思考使用哪个工具的agent）
3. ** 在对话中判断是否调用工具 & 调用哪些工具 — 做法与设计要点**
    
    让 Agent 在聊天流里自动判断并安全执行，常见模式：
    
    - **两阶段决策（推荐）**
        1. **Need?** 先由一个快速意图分类器（可以是小模型，也可以是 LLM 的短 prompt）回答“仅靠文本回答/需外部数据/需动作执行（写文件、run code、scrape、call API）”。
        2. **Which & How?** 如果需要调用，对话式 planner（LLM）输出工具序列（例如 JSON：`[{tool: "sheet_query", args: {...}}, {tool: "chart_gen", args:{...}}]`），并返回给 orchestrator 执行；执行结果回写对话上下文并返回给用户。
    - **工具调用提示（Tool-aware prompting）**
        
        在 prompt 里给 LLM 明确的工具能力说明和 few-shot 例子，让模型自己产出“tool plan”。对复杂场景可加入“chain of tools”示例，示范何时组合工具。
        
    - **约束与保护**
        - 把高风险操作（付款、删除）标记为需要**强交互确认**或人工审批。
        - 对外部 API 的输出做**校验器**（schema 校验、重复校验、可信度评分）。
        - 日志必须能回放（便于查错与合规）。
    - **避免幻觉**
        - 对需要事实/数据的回答优先走 RAG + 实时工具查询；若不能确定则显式告知“我需要访问 X 才能确认，是否允许？”而不是凭空断言。

## 📖 验收台 - Dashboard 优先级：🌟🌟

自动获取固定化流程

- 比如 运营每天都必须要重复看的bashboard或者数据有哪些？追踪活动数据？追踪竞品/热点数据？又或者说当前版本/活动 的多语言翻译进度 - 是否可以把这些需要固定经常看的内容，通过可视化的方式显示在验收台。以及在chat页面 AI生成的可用的素材/报告 - 比如 evet planner 活动策划方案，活动banner， kv，等，可以放在验收台

## 🪄 指令中心 - workflow 优先级：🌟🌟🌟

通过自然语言，排列组合Action 形成预设workflow

- 输入workflow 名词，功能描述；然后通过人工手动输入任务描述。例如下方示例，每一个步骤 都作为一个动作，通过用自然语言排列这些动作的顺序，让AI知道完成这个workflow 需要根据什么顺序使用哪些动作
    
    1. 搜索竞品游戏的最新资讯和数据
    
    2. 分析竞品的核心玩法和特色功能
    
    3. 收集用户对竞品的评价和反馈
    
    4. 对比我们产品与竞品的优劣势
    
    5. 生成详细的竞品分析报告
    

## 🏠 知识库 - knowledge Library 优先级：🌟🌟🌟

用户可以本地上传报告，文件到知识库，知识库权限分为 我的，公开，项目，技能 - 每个权限都允许不同的用户访问权限

知识库页面允许RAG搜索，或是支持在Chat page查询知识库内容。在动作库里，也支持 “查询某个知识库”作为一个动作

## 🔧 动作库 - Action 优先级：🌟🌟🌟🌟

最小颗粒度的工具库，储存当前产品可用所有功能

- 类似于 Dify 的Tool，alpha intelligence的marketplace，用于储存所有可用到的，可执行的工具。其作用是可以让Chat page中的Agent 来进行调用
